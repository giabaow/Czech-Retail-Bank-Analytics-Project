# 📊 Czech Retail Bank Analytics Project  

**Tools:** Power BI, Excel, SQL, Python (optional for preprocessing)  

---

## 📌 Project Overview  

This project simulates **real-world banking analytics** using a fictitious dataset of a Czech retail bank (as of December 20, 1998). The goal is to analyze client, account, loan, transaction, and regional demographic data to generate **actionable business insights** and support **data-driven decision-making**.  

The dataset includes:  
- **Clients:** Contact info, birth numbers (gender & date of birth), and account ownership  
- **Accounts & Dispositions:** Account details and ownership type  
- **Loans & Standing Orders:** Loan issuance, repayment, and recurring transactions  
- **Cards:** Issued card types for clients  
- **Regions & Districts:** Demographic and organizational data  
- **Targets:** Regional profitability indicators  
- **Transactions:** Daily banking transaction records  

---

## 🛠️ Tools & Technologies  

- **Power BI:** Data modeling, dashboard creation, report automation, and visual analytics  
- **Excel & SQL:** Data preprocessing, transformation, and verification  
- **Python (optional):** Data cleaning and supporting calculations  
- **Data Analytics Skills:** Trend analysis, KPI monitoring, demographic segmentation, and performance reporting  

---

## 🔎 Key Analyses & Insights  

- Built **star-schema data models** for loans, repayments, and all transactions to structure the data for reporting  
- Developed **Power BI dashboards** to monitor:  
  - Regional performance against **payment-for-statement income targets**  
  - Differences in loan amounts, standing orders, and card types across **age groups**  
  - Loan repayment behavior and organizational unit performance  
  - Transaction value distribution and demographic-adjusted regional insights  
- Created **maps and visualizations** to illustrate regional performance and trends  
- Generated **actionable insights** for decision-making, simulating an **enterprise analytics environment**
